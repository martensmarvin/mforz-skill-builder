Stap 1 van 6

Wat doet deze skill?

Beschrijf welke Zoho-taak je Claude wilt leren. Een skill is een instructie-set die Claude leert wanneer en hoe hij een specifiek Zoho-proces moet uitvoeren.

💡 Voorbeeld van een goede skill-definitie
"Deze skill leert Claude hoe hij leads van LinkedIn moet toevoegen aan Zoho CRM, inclusief bedrijfsverrijking en sales rep toewijzing."

Dit label zie jij in Claude, niet je team. Bijvoorbeeld: "zoho-lead-intake" of "crm-contact-enrichment"

Beschrijf in 2-3 zinnen wat Claude moet leren. Wees specifiek over de Zoho-acties. Bijvoorbeeld:

"Wanneer ik een LinkedIn-lead deel, moet deze skill Claude leren:
• Het bedrijf opzoeken in onze CRM-database
• Bedrijfsdata ophalen via externe enrichment API
• Een nieuw lead-record aanmaken in Zoho CRM
• De lead toewijzen aan de juiste sales rep o.b.v. regio
• Een follow-up taak aanmaken voor die rep"

Dit helpt Claude begrijpen wanneer de skill-uitvoering compleet is

Bijvoorbeeld: "Bevestiging met lead-ID, bedrijfsnaam, aantal medewerkers, toegewezen sales rep, en aanbevolen vervolgactie"

💡 Waarom dit belangrijk is
Een heldere skill-definitie is het verschil tussen een skill die Claude soms begrijpt en een skill die altijd werkt.

Vage skill-doelen → Claude raadt wat je bedoelt
Heldere skill-doelen → Claude weet precies wat te doen
📚 Wat je hier leert
Het definiëren van goede skills is de basis van effectieve Claude-automatisering. Deze skill-thinking kun je toepassen op elk Claude-project.
Stap 2 van 6

Hoe herkent Claude deze skill?

Triggers zijn de zinnen die deze skill activeren. Leer Claude herkennen wanneer jij deze specifieke skill wilt gebruiken.

📝 Wat zijn goede triggers?

Goede triggers = skill die altijd werkt
Slechte triggers = skill die soms werkt

✅ Goede triggers zijn:

  • Natuurlijk — zoals je het echt zou zeggen
  • Specifiek — duidelijk voor deze skill
  • Gevarieerd — meerdere manieren om hetzelfde te vragen

❌ Vermijd:

  • Te algemeen — "help me met CRM"
  • Te technisch — "voer CREATE_LEAD functie uit"
  • Slechts 1 variatie — meer is beter

💡 Hoeveel triggers heb je nodig? Minimaal 3-5 verschillende manieren. Meer triggers = Claude herkent de skill beter.

Voorbeelden (klik om toe te voegen):
maak een nieuwe lead aan
voeg dit bedrijf toe aan CRM
nieuwe prospect registreren
lead toevoegen met deze gegevens
📚 Wat je hier leert
Trigger-design is cruciaal voor skill-herkenning. Een skill met 5+ gevarieerde triggers werkt betrouwbaarder dan een skill met 2 algemene triggers. Dit leermoment geldt voor alle Claude skills, niet alleen Zoho.
Stap 3 van 6

Welke Zoho-toegang heeft deze skill nodig?

Skills verbinden Claude met Zoho via Connectors. Hier bepaal je welke Zoho-services en -acties deze specifieke skill mag gebruiken.

🔌 Hoe skills toegang krijgen tot Zoho

Skills op zichzelf geven Claude geen toegang — ze definiëren alleen WELKE toegang nodig is. De daadwerkelijke connectie maak je in Claude's settings door een Zoho Connector te activeren.

Deze skill bepaalt:

  • Welke Zoho-service (CRM, Books, Mail, etc.)
  • Welke acties (lezen, schrijven, zoeken, etc.)
  • Welke veiligheidsregels gelden

Vink aan als deze skill gegevens moet opzoeken, records moet aanmaken, of e-mails moet versturen in Zoho.

Kies de service die bij je skill past. Je ziet alleen wat je nodig hebt.

💡 Tip: Selecteer alleen wat echt nodig is. Meer permissions = groter risico bij fouten.

💡 Waarom skill-permissions belangrijk zijn
Te brede permissions → Risico dat skill onbedoeld data wijzigt
Te smalle permissions → Skill kan z'n taak niet uitvoeren

Goede skills krijgen precies genoeg toegang, niet meer.
📚 Wat je hier leert
Skill-based access control is een core concept in Claude-automatisering. Door hier bewust over na te denken, bouw je veilige én functionele skills.
Stap 4 van 6

Wat zijn de uitvoeringsstappen?

Skills bevatten logica. Beschrijf stap-voor-stap wat Claude moet doen wanneer deze skill wordt getriggerd.

🎯 Goede skill-logica is expliciet
Claude is slim, maar niet helderziend. Hoe meer je expliciet maakt, hoe betrouwbaarder je skill werkt.

Gebruik genummerde stappen met werkwoorden. Bijvoorbeeld:

1. Vraag gebruiker om lead-data (naam, email, bedrijf, rol)
2. Valideer dat verplichte velden (email, bedrijf) ingevuld zijn
3. Zoek in Zoho CRM of dit bedrijf al bestaat
4. Als nieuw bedrijf: haal bedrijfsdata op via Clearbit API
5. Maak lead-record aan in CRM met alle data
6. Bepaal verantwoordelijke sales rep o.b.v. bedrijfsregio
7. Maak follow-up taak voor die sales rep (deadline: +2 dagen)
8. Bevestig aan gebruiker: lead-ID, toegewezen rep, volgende stap

💡 Pro tip: Specificeer voorwaarden ("Als X, dan Y"). Dit maakt je skill robuust voor verschillende scenario's.

Elke productie-skill heeft edge case handling. Denk aan:

• Ontbrekende data — "Als email ontbreekt, vraag gebruiker erom"
• Duplicaten — "Als bedrijf bestaat, vraag: nieuwe lead of update contact?"
• API-fouten — "Als enrichment faalt, maak lead met beschikbare data + flag"
• Geen match — "Als geen sales rep voor regio, wijs toe aan [manager name]"

✅ Concrete instructies: "Als X, doe Y"
❌ Vage instructies: "Handle errors appropriately"

📚 Wat je hier leert
Edge case handling is wat demo-skills van productie-skills onderscheidt. Skills zonder edge cases werken in 80% van gevallen. Skills met edge cases werken in 95%+ van gevallen. Deze skill-design pattern geldt voor alle Claude-automatisering.
Stap 5 van 6

Welke veiligheidsregels gelden?

Elke skill heeft governance-regels. Bepaal hoe autonoom deze skill mag werken en welke acties altijd bevestiging vereisen.

🛡️ Skill governance = balans tussen snelheid en veiligheid
Te streng → Claude vraagt te vaak toestemming, skill vertraagt
Te los → Risico op onbedoelde acties, vooral bij nieuwe skills

💡 Best practice voor nieuwe skills: Test altijd eerst in Sandbox. Pas na succesvolle tests activeer je de skill in Productie.

📚 Wat je hier leert
Skill governance is geen "one size fits all". Verschillende skills hebben verschillende veiligheidsbehoeften. Goede skill designers vragen zich af: Wat is het risico? Hoe vaak wordt deze skill uitgevoerd? Zijn acties omkeerbaar? Deze risk-based thinking geldt voor alle Claude skills met write-access.
